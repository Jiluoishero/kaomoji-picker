# 项目状态

## 当前技术栈

- 运行平台：Windows
- UI：PySide6 / Qt 原生窗口
- 全局快捷键：Windows `RegisterHotKey`
- 剪贴板与粘贴：`pyperclip` + Win32 模拟 `Ctrl+V`
- 数据存储：`data.json`
- 配置存储：运行时生成的 `config.json`
- 系统托盘：`QSystemTrayIcon`

## 当前目标

这个项目已经从 `pywebview + WebView2` 迁移到 PySide6。迁移的目的不是保留旧实现，而是让窗口行为更接近软键盘：

- 浮动面板
- 圆角无边框窗口
- 系统托盘常驻
- 全局快捷键唤出
- 多屏定位
- 标题栏拖动
- 窗口大小保存
- 点击符号后粘贴到唤出前的窗口

## 当前文件结构

```text
main.py            # Qt 应用入口、窗口、热键、托盘
clipboard_util.py  # 剪贴板保存/恢复、模拟粘贴
data_manager.py    # 符号数据读写
config_manager.py  # 配置读写
data.json          # 符号数据
config.json        # 用户配置
requirements.txt   # Python 依赖
start.bat          # 一键启动脚本
```

## 已移除

- pywebview
- WebView 前端目录
- keyboard 全局 hook
- pystray 托盘实现

## 仍需人工验证

- 实机点击符号时是否始终粘贴到目标输入框
- 面板点击是否保持当前输入窗口焦点
- 标题栏拖动是否符合手感
- 大小调整与保存是否符合预期

## 发布前注意

- `data.json` 是默认表情数据，发布包需要保留。
- `config.json` 是用户本机配置，首次启动时自动生成，发布包不带。
- 不打包 `__pycache__/`、`*.log`、`build/`、`dist/`、`*.spec`。
- README 需要随功能变化同步，尤其是右键菜单管理、数据与配置说明。
