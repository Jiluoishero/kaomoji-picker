# 颜文字/符号快速输入工具 — 开发提示词

## 概述

开发一个 Windows 桌面小工具。按全局快捷键弹出一个浮动面板，展示颜文字和各种符号，点击后自动粘贴到当前活跃的应用中。支持分组管理、编辑模式（拖拽排序/删除）、批量添加。

## 技术栈

- **后端**: Python 3.10+
- **UI 渲染**: pywebview（使用系统 WebView2 引擎，不打包浏览器）
- **前端**: HTML + CSS + JavaScript（作为独立文件，不内嵌字符串）
- **全局快捷键**: keyboard 库
- **剪贴板**: pyperclip
- **窗口操作**: pywin32（win32gui / win32con）
- **数据存储**: JSON 文件（data.json + config.json）
- **系统托盘**: pystray
- **开机自启**: win32com.client（创建启动文件夹快捷方式）

依赖清单：
```
pywebview

pywin32
pystray
Pillow
```

## 功能需求

### 1. 全局快捷键触发

- 默认快捷键：`Ctrl+`` （反引号），可在设置中修改
- 按下后弹出浮动面板，出现在鼠标光标附近
- **单次模式**（按一次）：用户选取一个符号后，面板自动关闭
- **固定模式**（短时间内按两次，间隔 < 300ms）：面板保持打开，选取符号后不关闭，可连续选取多个。需要手动关闭（点关闭按钮或按 Esc）
- 面板已打开时按 Esc 关闭面板
- 非固定模式下，点击面板外部区域也关闭面板

### 2. 符号展示与选取

- 按**分组**展示，每个分组有名称
- 分组以**标签页**形式排列在顶部（水平，可滚动）
- 分组内的符号以**自适应宽度**的流式布局排列：
  - 每个符号格子宽度根据文字内容自适应（`width: auto`）
  - 符号文字两侧留有充足的内边距（`padding: 6px 12px`，或在符号文字两侧各添加一个空格字符），确保视觉舒适
  - 设置合理的 `min-width`（建议 36px），保证短符号（如 `♠`）也有足够的点击区域
  - 设置 `max-width`，防止极长颜文字溢出（超出时可适当缩小字号）
  - 整体使用 CSS `display: flex; flex-wrap: wrap; gap: 6px` 实现流式排列
- **点击符号后的完整流程**：
  1. 用 `win32gui.GetForegroundWindow()` 在弹出面板**之前**记录下当前活跃窗口句柄
  2. 保存当前剪贴板内容
  3. 将选中的符号写入剪贴板
  4. 用 `win32gui.SetForegroundWindow()` 将焦点切回之前记录的窗口
  5. 短暂延时（50ms）后模拟 Ctrl+V
  6. 再延时（100ms）后恢复原来的剪贴板内容
  7. 如果是单次模式，隐藏面板

### 3. 添加符号

- 顶栏右侧有 **"+"按钮**
- 点击后进入「添加模式」：
  1. 分组标签整体亮起（呼吸灯/脉冲高亮效果），提示用户选择目标分组
  2. 用户点击某个分组标签 → 该标签高亮锁定
  3. 标签栏下方就地展开一个多行文本框
  4. 每行输入一个符号，支持批量添加
  5. 点击确认按钮 → 写入数据 → 文本框收起，回到正常状态
  6. 点击取消或按 Esc 可退出添加模式

### 4. 编辑模式

- 在**设置界面**中可开启编辑模式
- 进入编辑模式后，类似手机桌面长按进入的「抖动」状态：
  - 符号格子上显示**删除角标**（右上角 ✕），点击可删除该符号
  - 分组标签可**拖拽排序**（拖拽后直接更新 JSON 中 `groups` 数组的索引顺序）
  - 分组标签上显示**删除按钮**，可删除整个分组（需二次确认）
  - **双击**分组标签可重命名
  - 分组标签栏末尾显示 **"+"按钮**，可添加新分组
- 退出编辑模式回到正常状态
- 拖拽排序使用前端原生 Drag and Drop API（`draggable` + `dragstart/dragover/drop`），不引入第三方库

### 5. 设置界面

- 顶栏右侧有**齿轮按钮 ⚙**，点击后切换到设置视图（或滑出抽屉）
- 设置项：
  - **快捷键配置**：显示当前快捷键，点击进入录制状态，按下新组合键即设定
  - **开机自启**：开关按钮
  - **编辑模式**：开关按钮，开启/关闭编辑模式
- 设置数据存储在独立的 `config.json` 中

### 6. 系统托盘（必需）

- 程序启动后缩小到系统托盘，不在任务栏显示主窗口
- 托盘图标右键菜单：
  - 显示面板
  - 退出程序
- 使用 pystray 库实现

## 数据存储

### data.json — 符号数据

```json
{
  "groups": [
    {
      "name": "开心",
      "items": [
        {"symbol": "(・∀・)"},
        {"symbol": "( ´∀`)"},
        {"symbol": "(*´∀`)"}
      ]
    },
    {
      "name": "悲伤",
      "items": [
        {"symbol": "(;´Д`)"},
        {"symbol": "( TдT)"}
      ]
    }
  ]
}
```

- 文件位置：与程序同目录下的 `data.json`
- 每次增删改（包括拖拽排序）后自动保存
- 首次运行若不存在，创建包含以上示例数据的默认文件
- **分组顺序 = 数组索引顺序**，拖拽排序直接操作数组元素位置（splice + insert）

### config.json — 应用配置

```json
{
  "hotkey": "ctrl+`",
  "auto_start": false,
  "double_click_interval": 300
}
```

- 文件位置：与程序同目录下的 `config.json`
- 首次运行若不存在，创建默认配置
- 修改设置后自动保存

## UI 设计要求

### 整体风格

- **深色主题**，配色参考 Discord / VSCode
- 背景色：`#1e1e2e`
- 卡片/分组区域背景：`#2d2d3d`
- 强调色/选中态：柔和的蓝紫色 `#7c6fef`
- 文字主色：`#cdd6f4`
- 文字次要色：`#a6adc8`
- 边框色：`#45475a`
- 圆角统一 8-12px
- 适当的 box-shadow 营造层次感

### 窗口

- **无边框窗口**（frameless），pywebview 设置 `frameless=True`
- 窗口大小约 420×480px
- 窗口本身也要有圆角（通过 CSS `border-radius` + pywebview 透明背景实现）
- 置顶显示（`on_top=True`）

### 布局（从上到下）

1. **顶栏**：左侧显示当前模式（"单次" / "固定📌"），右侧按顺序：添加按钮（+）、齿轮按钮（⚙）、关闭按钮（✕）
2. **分组标签页**：水平排列，可横向滚动，当前选中的标签有下划线高亮
3. **符号区域**：占据主要空间，可纵向滚动，符号以自适应宽度的流式布局排列

### 交互细节

- 符号格子 hover 时背景微微变亮，有 0.15s 的 transition
- 点击时有短暂的缩放反馈（scale 0.95 → 1.0）
- 面板出现时有轻微的 fadeIn + scaleUp 动画（0.12s）
- 面板消失时有轻微的 fadeOut 动画
- 分组标签切换有平滑过渡
- 滚动条自定义样式（细、半透明）
- **编辑模式**下符号/分组有轻微的抖动动画（CSS `@keyframes` 微旋转），表示可编辑状态
- **添加模式**下分组标签有呼吸灯/脉冲高亮效果

### pywebview JS-Python 桥接

创建一个 API 类暴露给前端：

```python
class Api:
    def get_groups(self):
        """返回所有分组数据"""
    
    def add_items(self, group_name: str, items_text: str):
        """批量添加符号，items_text 是多行文本，每行一个符号"""
    
    def delete_item(self, group_name: str, symbol: str):
        """删除指定符号"""
    
    def add_group(self, name: str):
        """添加新分组"""
    
    def delete_group(self, name: str):
        """删除分组"""
    
    def rename_group(self, old_name: str, new_name: str):
        """重命名分组"""
    
    def reorder_groups(self, group_names: list):
        """按传入的名称顺序重新排列分组（拖拽排序用）"""
    
    def paste_symbol(self, symbol: str):
        """执行剪贴板写入 + 切换窗口 + 模拟粘贴的完整流程"""
    
    def get_mode(self):
        """返回当前模式：'single' 或 'pinned'"""
    
    def close_panel(self):
        """隐藏面板"""
    
    def get_config(self):
        """返回当前配置"""
    
    def set_config(self, key: str, value):
        """更新配置项"""
    
    def set_hotkey(self, hotkey: str):
        """设置新的全局快捷键（注销旧的，注册新的）"""
    
    def set_auto_start(self, enabled: bool):
        """设置开机自启（创建/删除启动文件夹快捷方式）"""
```

前端通过 `window.pywebview.api.xxx()` 调用，所有方法返回值通过 Promise 获取。

### 窗口管理

- 不要每次都创建/销毁窗口，而是**创建一次，之后用 show/hide 切换**
- `keyboard` 库在独立线程中监听快捷键
- 快捷键触发时：
  1. 记录当前活跃窗口句柄
  2. 获取鼠标位置，计算面板弹出坐标（确保不超出屏幕边界）
  3. 移动窗口到目标位置
  4. 显示窗口
- 双击检测逻辑：记录上次按键时间戳，两次间隔 < 300ms 判定为双击，进入固定模式

### 面板关闭机制

- **Esc 关闭**：由前端 JS `keydown` 事件监听处理，调用 `pywebview.api.close_panel()`。不通过 `keyboard` 库的全局热键，避免与其他应用的 Esc 行为冲突。
- **非固定模式点击外部关闭**：后端使用定时轮询 `GetForegroundWindow()`（间隔 200ms），当检测到前台窗口不再是自己时，自动隐藏面板。轮询仅在面板可见且为单次模式时启动，面板隐藏后停止轮询。

### 关于线程安全

- `keyboard` 的回调运行在独立线程中
- pywebview 的窗口操作必须在主线程
- 使用 `webview.windows[0].evaluate_js()` 或事件队列来跨线程通信

### 开机自启实现

- 使用 `win32com.client.Dispatch('WScript.Shell')` 创建 `.lnk` 快捷方式
- 快捷方式路径：`%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\KaomojiPicker.lnk`
- 开启时创建快捷方式，关闭时删除

## 注意事项

1. 颜文字包含大量特殊 Unicode 字符（全角括号、特殊符号如 ∀ д ω 等），前端 HTML 必须声明 `<meta charset="UTF-8">`，JSON 读写必须指定 `encoding='utf-8'`
2. `SetForegroundWindow` 在 Windows 上有限制（前台锁定），可能需要用 `AttachThreadInput` 技巧绕过
3. 剪贴板恢复时，如果原来的内容是图片或富文本，pyperclip 只能处理纯文本。可以用 win32clipboard 更精细地保存/恢复，但作为 MVP 先只处理文本即可
4. pywebview 窗口的 `frameless=True` + 圆角需要同时设置 `transparent=True` 并在 CSS 中给 body 设置 `border-radius` 和 `overflow: hidden`
5. 确保窗口弹出位置不超出屏幕边界——获取屏幕分辨率，如果鼠标在右下角则向左上偏移
6. 编辑模式的拖拽排序使用前端原生 Drag and Drop API（`draggable` + `dragstart/dragover/drop`），不引入第三方库
7. 自适应宽度的符号格子需设置合理的 `min-width`（建议 36px）确保短符号有足够的点击区域
